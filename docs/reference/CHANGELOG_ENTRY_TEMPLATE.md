# Changelog Entry Template

Copy into `CHANGELOG.md` under `## [Unreleased]`:

```md
### Added
- <new feature or capability>

### Changed
- <behavior change or improvement>

### Deprecated
- <feature marked for removal>

### Removed
- <feature removed>

### Fixed
- <bug fix and user impact>

### Security
- <security fix or hardening change>
```

## Guidance

- Keep only headings that have at least one bullet.
- Use plain language focused on user-visible impact.
- Mention relevant paths or components when it improves clarity.
