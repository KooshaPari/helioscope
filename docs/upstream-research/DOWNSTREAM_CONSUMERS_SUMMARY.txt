ECOSYSTEM BREAKDOWN (6 MAJOR CATEGORIES)
================================================================================

1. IDE INTEGRATIONS (20+ projects)
   ├─ VS Code / Cursor / Windsurf
   │  └─ Pattern: TypeScript SDK, LSP, MCP server mode
   │  └─ Status: ✓ SDK exists; need MCP server mode
   │
   ├─ Neovim / Vim (Copilot.vim, ChatGPT.nvim)
   │  └─ Pattern: stdin/stdout JSONL, subprocess mode
   │  └─ Status: ✓ Works; need stable protocol guarantee
   │
   └─ JetBrains IDEs (IntelliJ, PyCharm, WebStorm, etc. — 40% of developers)
      └─ Pattern: LSP server, plugin API
      └─ Status: ✗ Missing; would need JetBrains plugin SDK

   CRITICAL FEATURE: MCP Server Mode (`helios --mcp`)
   EFFORT: 2-3 weeks | IMPACT: HIGH (unlocks 3+ IDE platforms)

KEY TECHNICAL REQUIREMENTS SUMMARY
================================================================================

AUTHENTICATION & SECRETS
├─ ✓ API key support (OPENAI_API_KEY, CODEX_API_KEY)
├─ ✓ ChatGPT sign-in support
├─ ? OAuth2 / PKCE support (needed for multi-tenant platforms)
└─ ? API key rotation handling (needed for long-running services)

SUBPROCESS INTERFACE
├─ ✓ stdin/stdout JSONL event streaming
├─ ✓ Image input support (--image)
├─ ? Config file + CLI override support (--config key=value)
└─ ? Clear exit code semantics (needed for automation)

├─ ✓ Sandbox workspace writes (optional)

├─ ? Network firewall config (--allowed-urls)
└─ ? Audit logging (JSON audit trail output)

├─ ? Response caching

├─ ? Latency metrics output

COMPETITIVE LANDSCAPE
================================================================================

INCUMBENT: GitHub Copilot
├─ Adoption: Industry standard (millions of users)
├─ Weakness: Limited to autocomplete, not full task automation
└─ helios-cli advantage: Full CLI agent with file I/O, bash execution

EMERGING COMPETITORS: Cursor, Windsurf, Aider
├─ Position: IDE-first, full-file reasoning
├─ Strength: Integrated UX, repository context
└─ helios-cli advantage: Language-agnostic CLI, composable with other tools

CODE REVIEW BOTS: Qodo, CodeRabbit
├─ Position: PR-focused, architecture-aware
├─ Strength: Multi-repo context, governance
└─ helios-cli opportunity: Could power their specialized analysis layer

ORCHESTRATORS: LangGraph, CrewAI, AutoGen
├─ Position: Coordinate multiple agents/tools
├─ Strength: Flexible, language-agnostic
└─ helios-cli opportunity: Be the default "coding agent" tool in these systems
