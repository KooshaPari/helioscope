## 6. Maintainability Issues & Risks

### Critical Issues
1. **Two Megafiles (9K+ LOC each)**
   - `codex.rs` – Core execution types/logic
   - `chat_composer.rs` – TUI input handler
   - **Impact:** High cyclomatic complexity, difficult refactoring, merge conflict risk
   - **Action:** Split into modules (stable_solution, not quick_fix)

2. **Bazel/Cargo Drift**
   - debug-client is workspace member but not buildable via Bazel
   - **Impact:** CI may fail on Bazel-only builds; inconsistent tooling experience
   - **Action:** Add missing BUILD.bazel file

3. **Unwrap() Calls (1,371 instances, 142 files)**
   - Significant panic risk in production paths
   - **Impact:** Silent failures converted to crashes
   - **Action:** Audit high-risk areas; replace with Result propagation

### High Priority Issues
4. **Code Markers (185 TODO/FIXME/HACK tags)**
   - Concentrated in tests (compact, delegate, exec_policy suites)
   - Some related to pending upstream PRs
   - **Action:** Consolidate into tracking system or tickets

5. **PROJECT-wtrees Proliferation**
   - 11 worktrees active (lane-a through lane-e, chore fixes, merge branches)
   - **Impact:** Potential stale branches; directory clutter
   - **Action:** Audit completion status; archive merged/abandoned branches

6. **Git Uncommitted Changes**
   - `.github/hooks/` directory untracked
   - `docs/reports/` directory untracked
   - **Action:** Review for intentional skip or add to .gitignore

### Medium Priority Issues
7. **TypeScript Distribution**
   - Small footprint (30.5K LOC across 5K files)
   - Heavily generated (app-server-protocol schema)
   - Limited test coverage (32 test files)
   - **Action:** Ensure schema generator is deterministic; validate CI pipeline
